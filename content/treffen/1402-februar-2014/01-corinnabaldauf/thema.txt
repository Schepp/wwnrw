Text: "Oje, der Code ist ja schlimm. Wer hat den Mist denn verbrochen?"
*klick*klick*tippitytipp* … "Oh. Ich. Hups. Dann kann ich nicht mal wen fragen, was das Gekröse hier macht… Mist!"

Code lesen ist oft mühsam. Sogar wenn man ihn selbst geschrieben hat - es muss nur genügend lange her sein. Das ist suboptimal, wenn man bedenkt, dass 80% der Lebensdauer einer Software auf den Wartungszeitraum entfallen. Code wird also viel öfter gelesen als geschrieben. Was tun?

Die Clean Code Bewegung arbeitet daran, dass wir Entwickler Verantwortung übernehmen und lesbaren, wartbaren und erweiterbaren Code schreiben.

Anstatt alle Methoden aufzuzählen, geht Corinna auf einige konkrete Beispiele ein und zeigt wann und wie man Code verbessern kann.
